## 一、项目概述

### 1.1 项目背景

智能学习管理助手是一款面向5-15岁孩子和家长的家庭教育SaaS工具，通过游戏化激励（积分/成就/愿望清单）和数据可视化，帮助孩子提升自驱力，帮助家长轻松掌握成长进度。

### 1.2 项目目标

- **短期目标（3个月）**：完成MVP版本开发，支持3-5组家庭内测
- **中期目标（6个月）**：产品正式上线，积累100+活跃用户
- **长期目标（12个月）**：完善产品生态，探索商业模式

### 1.3 项目里程碑

| 里程碑 | 时间 | 交付物 |
|--------|------|--------|
| 迭代一完成 | Week 4 | MVP版本（计划+打卡+积分） |
| 迭代二完成 | Week 8 | 体验版（+统计+愿望+成就） |
| 迭代三完成 | Week 12 | 完整版（+习惯+AI+同步） |
| 正式上线 | Week 13-14 | 生产环境部署、应用商店上架 |

## 四、开发规范

### 4.1 代码规范

#### Git 工作流

```
main (生产分支)
  ↑
develop (开发分支)
  ↑
feature/xxx (功能分支)
  ↑
hotfix/xxx (热修分支)
```

**分支命名**
- 功能分支：`feature/功能名称`（如：feature/user-login）
- 修复分支：`fix/问题描述`（如：fix/login-error）
- 热修分支：`hotfix/问题描述`

**提交规范**
```
<type>(<scope>): <subject>

<body>

<footer>
```

Type 类型：
- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

示例：
```
feat(auth): 添加用户登录功能

- 实现手机号登录
- 实现JWT认证
- 添加登录页面

Closes #123
```

#### 前端代码规范

**命名规范**
- 组件：PascalCase（如：UserProfile.vue）
- 文件：kebab-case（如：user-service.ts）
- 变量：camelCase（如：userName）
- 常量：UPPER_SNAKE_CASE（如：MAX_COUNT）

**Vue 组件规范**
```vue
<template>
  <!-- 模板 -->
</template>

<script setup lang="ts">
// 1. imports
import { ref, computed } from 'vue'
import UserService from '@/services/user-service'

// 2. types
interface User {
  id: string
  name: string
}

// 3. props & emits
const props = defineProps<{
  userId: string
}>()

const emit = defineEmits<{
  (e: 'update', user: User): void
}>()

// 4. reactive state
const user = ref<User | null>(null)
const loading = ref(false)

// 5. computed
const userName = computed(() => user.value?.name ?? '')

// 6. methods
async function fetchUser() {
  loading.value = true
  try {
    user.value = await UserService.getById(props.userId)
  } finally {
    loading.value = false
  }
}

// 7. lifecycle
onMounted(fetchUser)
</script>

<style scoped>
/* 样式 */
</style>
```

#### 后端代码规范

**模块结构**
```typescript
// 控制器
@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}
  
  @Get(':id')
  async findOne(@Param('id') id: string): Promise<User> {
    return this.usersService.findOne(id)
  }
}

// 服务
@Injectable()
export class UsersService {
  constructor(
    @InjectRepository(User)
    private readonly userRepository: Repository<User>
  ) {}
  
  async findOne(id: string): Promise<User> {
    const user = await this.userRepository.findOne({ where: { id } })
    if (!user) {
      throw new NotFoundException('User not found')
    }
    return user
  }
}
```

### 4.2 代码审查

**审查流程**
1. 开发者提交PR
2. 指定审查人（至少1人）
3. 审查人提出意见
4. 开发者修改
5. 审查通过，合并代码

**审查清单**
- [ ] 代码符合规范
- [ ] 功能实现正确
- [ ] 有必要的注释
- [ ] 无明显的性能问题
- [ ] 无安全漏洞
- [ ] 有对应的测试

### 5.1 任务管理

**任务拆分原则**
- 一个任务不超过2天
- 任务有明确的验收标准

**任务状态**

- 测试中（Testing）

```
  /--------\
```

| 测试类型 | 时间点 | 负责人 | 工具 |

| 环境 | 用途 | 数据 |

### 6.3 Bug 管理

**Bug 分级**
| 级别 | 定义 | 响应时间 | 解决时间 |
|------|------|----------|----------|
| P0 | 系统崩溃，核心功能不可用 | 立即 | 4小时 |
| P1 | 主要功能异常，有 workaround | 2小时 | 1天 |
| P2 | 次要功能异常 | 1天 | 3天 |
| P3 | UI问题，优化建议 | 3天 | 迭代内 |

**Bug 流程**
```
发现Bug → 提交Bug → 分配Bug → 修复Bug → 验证Bug → 关闭Bug
```
